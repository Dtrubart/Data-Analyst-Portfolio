**Hierarchical Modeling Using M (Power Query Language)**

## 🎯 Objective

Transform a flat employee–manager dataset into an analytics-ready hierarchical model using Power Query.

Outputs:

- Full Reporting Hierarchy
- Direct Reports count
- Total Reports count

## 🧾 Input Data

**Source file:** `OfficeSpace.csv`

| Column | Description |
|--------|------------|
| Employee Name | Employee identifier |
| Manager Name  | Direct manager (blank for top-level employees) |

# 🏗 Solution Architecture

The solution follows five logical layers:

1. Data Cleaning & Normalization  
2. Hierarchy Construction (Functional Recursion)  
3. Direct Reports Aggregation  
4. Total Reports Aggregation  
5. Final Output Shaping  

---

# 1️⃣ Data Cleaning & Normalization

### Key Transformations

- Trim whitespace
- Remove null/blank employees
- Convert empty manager values to `null`


Normalized = Table.TransformColumns(
    Clean,
    {{"Manager Name", each if _ = "" then null else _, type nullable text}}
)

Clean data ensures reliable hierarchical joins.
---

2️⃣ Hierarchy Construction (Functional Approach)

Power Query simulates recursion using List.Generate.

Helper Function — GetManager
GetManager = (emp as text) as nullable text =>
    let
        match = Table.SelectRows(Normalized, each [Employee Name] = emp),
        mgr = if Table.RowCount(match) = 0 then null else match{0}[Manager Name]
    in
        mgr
Recursive Simulation — GetAncestors
GetAncestors = (emp as text) as list =>
    let
        ancestors =
            List.Generate(
                () => [cur = emp, mgr = GetManager(emp)],
                each [mgr] <> null,
                each [cur = [mgr], mgr = GetManager([mgr])],
                each [mgr]
            )
    in
        ancestors
How It Works

Start at employee

Move upward through managers

Stop when manager is null

Return list of ancestors

Build Reporting Hierarchy
Text.Combine(
    List.Combine({List.Reverse([Ancestors]), {[Employee Name]}}),
    " > "
)
---

3️⃣ Direct Reports
DirectCountsRaw = Table.Group(
    Normalized,
    {"Manager Name"},
    {{"Direct Reports", each Table.RowCount(_), Int64.Type}}
)

Groups by manager and counts rows.

---

4️⃣ Total Reports (Direct + Indirect)
Strategy

Expand ancestor lists

Create employee–manager pairs

Group by manager

Count rows

ExpandedAnc = Table.ExpandListColumn(AncPairs, "Ancestors")

Then:

TotalCountsRaw = Table.Group(
    RenamedAnc,
    {"Manager"},
    {{"Total Reports", each Table.RowCount(_), Int64.Type}}
)

---

5️⃣ Final Output

Merge Direct Reports

Merge Total Reports

Replace nulls with 0

Sort by Total Reports

Final table is ready for Power BI visualization.

# 🖥 Alternative UI-Based Implementation (Step-Driven Self-Join Approach)

## Overview

In addition to the functional `List.Generate` solution, a second implementation was developed using a fully UI-driven approach inside Power Query.

This version avoids custom functions and instead builds the hierarchy through iterative self-joins.

---

## Methodology

The hierarchy is constructed by:

1. Duplicating the **Manager Name** column.
2. Performing successive self-joins to retrieve:
   - L1 (Direct Manager)
   - L2 (Manager’s Manager)
   - L3
   - L4
3. Expanding each join level.
4. Combining hierarchy levels into a single text path.
5. Grouping data to compute Direct and Total reports.

This approach simulates hierarchical expansion through repeated relational joins.

---

## Characteristics

| Dimension | Functional (List.Generate) | UI Self-Join Approach |
|------------|----------------------------|------------------------|
| Scalability | High (dynamic depth) | Limited to predefined levels |
| Maintainability | High | Moderate |
| Transparency | Requires M knowledge | Easy to follow visually |
| Performance | Efficient | Can degrade with many levels |
| Production Readiness | Strong | Suitable for small hierarchies |
