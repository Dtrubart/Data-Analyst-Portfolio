
## 2️⃣ Flatten the Stack — Excel Formula Solution

### Objective
Flatten nested JSON data stored in a column by expanding list-based attributes into individual rows while preserving the parent record context.

This exercise simulates a common analytics task where transactional data contains nested structures such as orders with multiple line items.

---

## Problem Description

The source dataset contains order-level records with a JSON field (`line_items`) representing multiple purchased products.

Each record looks like this:

| order_number | order_date | line_items | fulfillment |
|---|---|---|---|
| 387005 | 22/01/2016 | JSON array of products | In store |

Example of the nested JSON:

```json
[
  {
    "product": {
      "product_name": "MGS Age of Empires II Gold Edition2009 E172",
      "product_price": 32
    },
    "quantity": 3
  },
  {
    "product": {
      "product_name": "A. Datum Bridge Digital Camera M300 Pink",
      "product_price": 186.9
    },
    "quantity": 2
  }
]
```

The challenge is to transform this into a **flat transactional table with one row per product**.

---

## Expected Output

| order_number | order_date | fulfillment | product_name | product_price | quantity |
|---|---|---|---|---|---|
| 387005 | 22/01/2016 | In store | MGS Age of Empires II Gold Edition2009 E172 | 32 | 3 |
| 387005 | 22/01/2016 | In store | A. Datum Bridge Digital Camera M300 Pink | 186.9 | 2 |

---

## Solution Approach (Excel Only)

Because the JSON is stored as text inside Excel, the solution requires:

- Identifying how many items exist inside each JSON block
- Expanding each order into multiple rows
- Mapping each generated row back to its source order
- Extracting attributes from the JSON structure using text functions
- Cleaning the extracted values to handle hidden characters and locale differences

---

## Step 1 — Count Nested Items

Each product entry contains the key `"product_name"`.

By counting how many times this appears in the JSON text, we determine how many line items belong to each order.

```excel
=(LARGO([@line_items])-LARGO(SUSTITUIR([@line_items];"""product_name""";"")))/LARGO("""product_name""")
```

Example result:

| order_number | item_count |
|---|---|
| 387005 | 3 |
| 395005 | 2 |
| 423011 | 1 |

---

## Step 2 — Generate Flattened Row Index

The total number of rows needed for the flattened dataset is the **sum of all line items**.

| Line_ID|
|---|
| 1 |
| 2 |
| 3 |
| 4 |
| 5 |

Each row corresponds to one product line.

---

## Step 3 — Map Flattened Rows to Source Orders

Using cumulative item counts, each flattened row is mapped back to the order that generated it.

```excel
=COINCIDIRX(A2;sales_orders[acu_items];1)
```

Example mapping:

| flat_row_id | src_row |
|---|---|
| 1 | 1 |
| 2 | 1 |
| 3 | 1 |
| 4 | 2 |
| 5 | 2 |

This indicates which source order each flattened row belongs to.

---

## Step 4 — Determine Item Position Within Order

Each flattened row corresponds to the **nth item within its order**.

```excel
=A2-SI(B2=1;0;INDICE(sales_orders[acu_items];B2-1))
```

Example:

| flat_row_id | Item_Secuence | Line_level_item |
|---|---|---|
| 1 | 1 | 1 |
| 2 | 1 | 2 |
| 3 | 1 | 3 |
| 4 | 2 | 1 |

---

## Step 5 — Extract Product Name

The product name is extracted from the JSON text using `TEXTODESPUES` and `TEXTOANTES`.

```excel
=TEXTOANTES(TEXTODESPUES(G2;"""product_name"": """;C2);"""")
```

---

## Step 6 — Extract Quantity

```excel
=VALOR(ESPACIOS(SUSTITUIR(SUSTITUIR(TEXTOANTES(TEXTODESPUES(G2;"""quantity"": ";C2);"}");CARACTER(10);"");CARACTER(13);"")))
```

Cleaning steps are required because JSON fields may contain hidden newline characters.

---

## Step 7 — Extract Product Price

The value must therefore be converted before casting to numeric.

```excel
=ESPACIOS(SUSTITUIR(SUSTITUIR(TEXTOANTES(TEXTODESPUES(G2;"""product_price"": ";C2);"}");CARACTER(10);"");CARACTER(13);""))
```

---

## Key Data Cleaning Challenges

### Hidden Characters

JSON fields contained newline characters (`CHAR(10)` and `CHAR(13)`) that prevented standard `TRIM/ESPACIOS` cleaning.

These had to be removed explicitly using CHAR(10) and CHAR(13):

```excel
SUSTITUIR(texto;CARACTER(10);"")
```

---

## Final Output Structure

| order_number | order_date | fulfillment | product_name | product_price | quantity |
|---|---|---|---|---|---|

This structure represents a **fully normalized line-item transaction table suitable for analysis**.
