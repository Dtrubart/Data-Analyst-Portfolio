# Org Chart Overhaul - Excel Formula Solution  
**Dynamic Array Approach**

---

## 🎯 Objective

Reconstruct a full organizational hierarchy in Excel using **formulas only**, deriving:

- Full reporting chain (top → bottom)
- Direct reports count
- Total reports count (direct + indirect)

This solution demonstrates how modern Excel (Microsoft 365) can simulate hierarchical traversal without native recursion support.

---

## 🧾 Input Data

**Source file:** `OfficeSpace.csv`

| Column | Description |
|--------|------------|
| Employee Name | Employee identifier |
| Manager Name  | Direct manager (blank for top-level employees) |

The CSV is imported as an **Excel Structured Table**, enabling dynamic references and automatic recalculation.

---

## 🏗 Solution Architecture

Excel does not provide native recursion.  
This solution simulates hierarchical traversal using dynamic array functions:

- `LET` → Variable scoping and performance optimization  
- `SCAN` → Iterative upward traversal  
- `XLOOKUP` (`BUSCARX`) → Parent lookup  
- `FILTER` (`FILTRAR`) → Remove null values  
- `SORTBY` (`ORDENARPOR`) → Reverse hierarchy order  
- `TEXTJOIN` (`UNIRCADENAS`) → Concatenate chain  
- `COUNTIFS` (`CONTAR.SI.CONJUNTO`) → Direct & indirect aggregation  

All calculations are fully dynamic and update automatically when the dataset changes.

---

# 1️⃣ Reporting Hierarchy (Manager Drilldown)

## 🎯 Purpose

Generate the full chain of command for each employee, displayed from the highest-ranking manager down to the employee.

**Example Output**


CEO > VP Finance > Finance Manager > Analyst


---

## 🧠 Conceptual Logic

Since Excel cannot recurse, recursion is simulated using iterative evaluation:

1. Start from the employee.
2. Lookup the employee’s manager.
3. Repeat the lookup up to a defined depth.
4. Remove blanks.
5. Reverse the order.
6. Concatenate the results.

---

## 🧮 Formula

```excel
=LET(
  heir; SCAN([@[Employee Name]]; SECUENCIA(10);
       LAMBDA(acu;val;
           BUSCARX(acu; [Employee Name]; [Manager Name]; 0; 0)
       )
  );
  fil; FILTRAR(heir; heir<>0; "");
  ord; ORDENARPOR(fil; SECUENCIA(FILAS(fil)); -1);
  UNIRCADENAS(" > "; VERDADERO; ord;[@[Manager Name]];[@[Employee Name]])
)
__________________________________________________________
🔍 Step-by-Step Breakdown

1️⃣ SCAN — Iterative Traversal
SCAN([@[Employee Name]]; SECUENCIA(10); ...)

Starts with the employee.

Repeats lookup 10 times (depth limit).

Each iteration replaces the previous value with its manager.

Effectively simulates recursion.
__________________________________________________________
2️⃣ BUSCARX — Manager Lookup
BUSCARX(acu; [Employee Name]; [Manager Name]; 0; 0)

Returns the manager of the current employee.
Returns 0 when no manager exists (top-level reached).
__________________________________________________________
3️⃣ FILTRAR — Remove Null Results

Removes empty or zero values from traversal output.
__________________________________________________________
4️⃣ ORDENARPOR — Reverse Order

Reverses the array so the hierarchy displays top → bottom.
_________________________________________________________
5️⃣ UNIRCADENAS — Concatenate Hierarchy
UNIRCADENAS(" > "; VERDADERO; ...)

Joins all levels using a readable delimiter.
_________________________________________________________
⚠ Depth Limit
SECUENCIA(10)

Limits traversal to 10 levels to prevent infinite loops in case of circular references.
Increase this value if the organization has deeper structures.
_________________________________________________________
2️⃣ Direct Reports
🎯 Purpose

Count employees who report directly to the current employee.

🧮 Formula
=CONTAR.SI.CONJUNTO([Manager Name];[@[Employee Name]])
🧠 Logic

Counts how many times the employee appears in the Manager Name column.
_________________________________________________________
3️⃣ Total Reports (Direct + Indirect)
🎯 Purpose

Count all employees under the current employee in the hierarchy.

🧮 Formula
=CONTAR.SI.CONJUNTO(
   [Reporting Heirarchi (Manager drilldown)];
   "*"&[@[Employee Name]]&"*"
)-1
🧠 Conceptual Logic

If an employee’s name appears inside another employee’s hierarchy string,
then they are an ancestor.

Example:

CEO > VP Finance > Analyst

Since CEO appears inside the string, CEO is an ancestor of Analyst.

-1 removes self-counting.
