## 2.3.1 Power Query - Power Bi Solution

## Source Data Structure

The source file is a CSV with the following columns:

| Column | Description |
|---|---|
| order_number | Unique order identifier |
| order_date | Date of the order |
| line_items | JSON array containing product and quantity details |
| fulfillment | Fulfillment channel |

Example of `line_items`:

```json
[
  {
    "product": {
      "product_name": "MGS Age of Empires II Gold Edition2009 E172",
      "product_price": 32
    },
    "quantity": 3
  },
  {
    "product": {
      "product_name": "A. Datum Bridge Digital Camera M300 Pink",
      "product_price": 186.9
    },
    "quantity": 2
  }
]
```

The goal is to transform each item inside the JSON array into its own row.

## Power Query M Code

```powerquery
let
    Origen = Csv.Document(File.Contents("D:\Data Examples\2. Flatten the Stack\sales_orders.csv"),[Delimiter=","]),
    #"Encabezados promovidos" = Table.PromoteHeaders(Origen, [PromoteAllScalars=true]),
    #"Tipo cambiado" = Table.TransformColumnTypes(#"Encabezados promovidos",{{"order_number", Int64.Type}, {"order_date", type date}, {"line_items", type text}, {"fulfillment", type text}}),
    #"JSON analizado" = Table.TransformColumns(#"Tipo cambiado",{{"line_items", Json.Document}}),
    #"Se expandió line_items" = Table.ExpandListColumn(#"JSON analizado", "line_items"),
    #"Se expandió line_items1" = Table.ExpandRecordColumn(#"Se expandió line_items", "line_items", {"product", "quantity"}, {"line_items.product", "line_items.quantity"}),
    #"Se expandió line_items.product" = Table.ExpandRecordColumn(#"Se expandió line_items1", "line_items.product", {"product_name", "product_price"}, {"line_items.product.product_name", "line_items.product.product_price"}),
    #"Columnas con nombre cambiado" = Table.RenameColumns(#"Se expandió line_items.product",{{"line_items.product.product_name", "product_name"}, {"line_items.product.product_price", "product_price"}})
in
    #"Columnas con nombre cambiado"
```

## Step-by-Step Transformation Logic

### Step 1 — Import the CSV File

```powerquery
Origen = Csv.Document(File.Contents("D:\Data Examples\2. Flatten the Stack\sales_orders.csv"),[Delimiter=","])
```

This step loads the raw CSV file into Power Query.

At this point, the dataset is still a raw table without proper headers or data types.

---

### Step 2 — Promote the First Row to Headers

```powerquery
#"Encabezados promovidos" = Table.PromoteHeaders(Origen, [PromoteAllScalars=true])
```

This converts the first row of the imported file into column headers.

The table now has meaningful field names such as:

- `order_number`
- `order_date`
- `line_items`
- `fulfillment`

---

### Step 3 — Assign Data Types

```powerquery
#"Tipo cambiado" = Table.TransformColumnTypes(#"Encabezados promovidos",{{"order_number", Int64.Type}, {"order_date", type date}, {"line_items", type text}, {"fulfillment", type text}})
```

This step defines the correct data types for each column:

- order_number → whole number
- order_date → date
- line_items → text
- fulfillment → text

Typing the columns early improves consistency and avoids transformation issues later in the query.

---

### Step 4 — Parse the JSON Text

```powerquery
#"JSON analizado" = Table.TransformColumns(#"Tipo cambiado",{{"line_items", Json.Document}})
```

This is the key transformation.

The `line_items` column originally contains JSON stored as text.  
`Json.Document` converts that text into a Power Query structured object.

After this step, each value in `line_items` becomes a **list** instead of plain text.

---

### Step 5 — Expand the JSON List into Rows

```powerquery
#"Se expandió line_items" = Table.ExpandListColumn(#"JSON analizado", "line_items")
```

Each order may contain multiple items inside the JSON array.

This step expands the list so that:

- one order with 3 items becomes 3 rows
- one order with 2 items becomes 2 rows

This is the core **flattening step**.

The parent columns (`order_number`, `order_date`, `fulfillment`) are repeated automatically for each generated row.

---

### Step 6 — Expand the Item-Level Record

```powerquery
#"Se expandió line_items1" = Table.ExpandRecordColumn(#"Se expandió line_items", "line_items", {"product", "quantity"}, {"line_items.product", "line_items.quantity"})
```

Each item in the expanded list is still a record with two fields:

- `product`
- `quantity`

This step expands those fields into columns.

At this stage:

- `quantity` becomes a standard column
- `product` remains a nested record

---

### Step 7 — Expand the Nested Product Record

```powerquery
#"Se expandió line_items.product" = Table.ExpandRecordColumn(#"Se expandió line_items1", "line_items.product", {"product_name", "product_price"}, {"line_items.product.product_name", "line_items.product.product_price"})
```

The `product` field itself contains another nested record.

This step expands it to extract:

- `product_name`
- `product_price`

After this step, the dataset is already mostly flat.

---

### Step 8 — Rename Final Columns

```powerquery
#"Columnas con nombre cambiado" = Table.RenameColumns(#"Se expandió line_items.product",{{"line_items.product.product_name", "product_name"}, {"line_items.product.product_price", "product_price"}})
```

This final cleanup step renames technical column names into cleaner business-friendly names:

- `line_items.product.product_name` → `product_name`
- `line_items.product.product_price` → `product_price`

This improves readability and makes the output easier to use in reporting or analysis.

---

## Final Output Structure

The final table contains one row per line item with the parent order context preserved.

| order_number | order_date | fulfillment | line_items.quantity | product_name | product_price |
|---|---|---|---|---|---|

You may optionally rename `line_items.quantity` to simply `quantity` as a final cleanup step.

Example final output:

| order_number | order_date | fulfillment | quantity | product_name | product_price |
|---|---|---|---|---|---|
| 387005 | 22/01/2016 | In store | 3 | MGS Age of Empires II Gold Edition2009 E172 | 32 |
| 387005 | 22/01/2016 | In store | 2 | A. Datum Bridge Digital Camera M300 Pink | 186.9 |
