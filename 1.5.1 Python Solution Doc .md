# Org Chart Overhaul - Python Solution  
**Hierarchical Modeling with Pandas + Graph Traversal**

---

## 🎯 Objective

Transform a flat employee–manager dataset into an analytics-ready organizational model using Python.

The solution generates:

- **Reporting Hierarchy** (Top → Employee)
- **Direct Reports**
- **Total Reports** (Direct + Indirect descendants)

Unlike Excel or SQL string matching approaches, this implementation models the organization as a **directed tree structure** and computes totals using dynamic programming.

---

## 🧾 Input Data

File: `OfficeSpace.csv`

| Column | Description |
|--------|------------|
| Employee Name | Employee identifier |
| Manager Name  | Direct manager (blank for top-level employees) |

The file is loaded using:

pd.read_csv("OfficeSpace.csv", dtype=str)

🏗 Solution Architecture

The solution is structured into six logical stages:

1) Data Cleaning & Normalization

2) Graph Construction (Manager → Children mapping)

3) Reporting Hierarchy Computation (Upward traversal)

4) Direct Reports Calculation

5) Total Reports Calculation (Postorder Dynamic Programming)

6) Final Output Formatting
---

1️⃣ Data Cleaning & Normalization
Purpose

Ensure reliable graph modeling before hierarchy computation.

Steps Performed

Trim whitespace

Remove empty employees

Remove accidental header rows

Normalize blank managers to NA

Enforce unique employee names
________________________________________________________________

df["Employee Name"] = df["Employee Name"].fillna("").str.strip()
df["Manager Name"]  = df["Manager Name"].fillna("").str.strip()

df = df[df["Employee Name"] != ""].copy()

df.loc[df["Manager Name"] == "", "Manager Name"] = pd.NA

df = df.drop_duplicates(subset=["Employee Name"], keep="first").copy()

Clean data ensures consistent parent-child relationships.
---

2️⃣ Graph Construction

The organization is modeled as a directed graph:

manager_of: employee → manager mapping

children: manager → list of direct reports

_________________________________________________________________

manager_of = df.set_index("Employee Name")["Manager Name"].to_dict()

children = defaultdict(list)
for emp, mgr in manager_of.items():
    if pd.notna(mgr):
        children[mgr].append(emp)

This transforms the flat table into an adjacency structure.
---

3️⃣ Reporting Hierarchy (Upward Traversal)
Purpose

Construct full reporting path for each employee.

Example output:

CEO > VP Finance > Finance Manager > Analyst
Implementation:

def reporting_hierarchy(emp: str, max_hops: int = 100) -> str:
    chain = [emp]
    seen = {emp}
    cur = emp

    for _ in range(max_hops):
        mgr = manager_of.get(cur, pd.NA)
        if pd.isna(mgr):
            break
        if mgr in seen:
            chain.append(f"[CYCLE:{mgr}]")
            break
        chain.append(mgr)
        seen.add(mgr)
        cur = mgr

    return " > ".join(reversed(chain))

Key Features

Iterative upward traversal

Cycle detection using seen set

Safety guard (max_hops)

Reverses chain to display Top → Employee

This avoids recursion stack issues and ensures robustness.
---

4️⃣ Direct Reports

Direct reports are computed directly from the adjacency list:

df["Direct Reports"] = df["Employee Name"].map(
    lambda e: len(children.get(e, []))
).astype(int)

Time complexity: O(n)
---

5️⃣ Total Reports (Postorder Dynamic Programming)
Purpose

Compute total descendants for each employee efficiently.


Strategy:

Instead of repeated traversal per node (O(n²)), the solution:

Computes out-degrees

Uses a queue (deque)

Processes leaf nodes first

Aggregates descendant counts upward

Algorithm:
total_desc = {emp: 0 for emp in manager_of.keys()}
out_degree = {emp: len(children.get(emp, [])) for emp in manager_of.keys()}
q = deque([emp for emp, deg in out_degree.items() if deg == 0])

while q:
    node = q.popleft()
    mgr = manager_of.get(node, pd.NA)
    if pd.isna(mgr):
        continue

    total_desc[mgr] = total_desc.get(mgr, 0) + (1 + total_desc.get(node, 0))

    out_degree[mgr] -= 1
    if out_degree[mgr] == 0:
        q.append(mgr)
Why This Is Efficient

This is a postorder traversal using dynamic programming.

Time Complexity: O(n)
Space Complexity: O(n)

Each node is processed exactly once.

Cycle Detection

After traversal:

if any(deg > 0 for deg in out_degree.values()):
    raise ValueError("Cycle detected (or invalid hierarchy).")

Ensures structural validity before final output.
